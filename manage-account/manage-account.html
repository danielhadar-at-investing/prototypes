Manage Account

Entry Points

  • New "Manage Account" item above Sign Out in the More menu
  • Tapping user name/avatar at top-left of More menu

Visibility

  • Registered users only (members + subscribers)
  • Guest/non-logged-in users don't see the item, and tapping their name doesn't do anything (same as today)

Profile Photo

  • Shows current photo or default placeholder
  • Tap opens native image picker (gallery)
  • App resizes/compresses before upload
  • If an invalid file is selected, it is rejected after a brief delay with an error alert - separate alerts for wrong type vs. oversized
Copy - wrong file type:
Title: "Invalid file type"
Message: "Please upload a JPG or PNG only"
Button: "Got it"
Copy - file too large:
Title: "Photo too large"
Message: "Please upload a photo smaller than 10 MB"
Button: "Got it"

Photo API response

  • Success: toast - "Photo successfully updated"
  • Failure: alert - Title: "Upload Failed" / Message: "Failed to upload photo, please try again"

Personal Information

  • First name, Last name, Display name - all editable
  • Email - read-only, shown as 4th row in the same card; rendered in a disabled/muted style with the social provider icon (Apple / Google / Facebook) to the right of the address
  • Pre-populated from user profile
  • Display name helper: "The name displayed on your comments"
  • Save button is disabled by default; enabled only when at least one field differs from the last saved value - re-disables if user manually restores the original content
  • Tapping Save replaces the button with an activity indicator until the operation completes, then shows a toast
  • On failure, the button is restored and a failure toast is shown

Name validation

  • First name / Last name: letters only, 1-30 chars. Save disabled if empty or invalid
  • Display name: letters and numbers only, 4-16 chars, no symbols or spaces. Save disabled if empty, invalid, or under 4 chars. Input blocked at 16 chars (no error)
  • One red error shown at a time below the card. Priority: First name > Last name > Display name. Errors: "First name should only include letters" / "Last name should only include letters" / "Display name should only include letters and numbers"
Copy - save success toast: "Your changes have been saved"
Copy - save failure alert: "Failed to save changes, please try again"

Login method

  • Users log in either via email/password or through a social provider (Apple, Google, or Facebook)
  • Social login users have no password associated with their account - the Login information section and Change Password option are not shown for them
  • The login method is determined at registration and cannot be changed from this screen

Email (Read-only)

  • Non-interactive, rendered in a disabled/muted style
  • No provider icon - email address is right-aligned text only

Change Password

  • Tapping "Change Password" opens a page sheet modal
  • Sheet has two fields: New password, Confirm password - both masked with eye toggle
  • Password requirements shown below the fields: "Your password must contain: / At least 2 letters / At least 2 numbers / Maximum 15 characters / Passwords match"
  • Each condition lights up green (with filled checkmark) only when met; gray otherwise
  • "Maximum 15 characters" turns green as soon as the field has 1+ character (valid range is 1-15)
  • "Passwords match" turns green only when both New password and Confirm password are non-empty and identical
  • Update Password button is disabled until new password and retype password are both non-empty and match
  • Enable Password AutoFill on the update password screen (new and retype fields)
  • Closing the sheet (X or backdrop) clears all fields and resets the password conditions to their unmet state
  • Tapping Update Password replaces the button with an activity indicator; on completion the sheet dismisses and a toast is shown
Copy - update password success toast: "Password updated"
Copy - update password failure toast: "Couldn't update password. Please try again"

Delete Account

  • Moved here from Settings - remove it from the Settings screen
  • Functionality stays identical to today's implementation
  • Displayed as a standalone white card with red (destructive) text at the bottom of the screen
